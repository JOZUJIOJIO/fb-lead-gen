## 1. 系统启动

### 环境要求

- Docker Desktop（已安装并运行）
- 至少 4GB 可用内存
- 稳定的网络连接

### 启动步骤

```bash
# 进入项目目录
cd "/Users/yewudao/Desktop/Tech Bridge/02_建造者/fb-lead-gen"

# 首次启动（构建 + 启动）
docker compose up -d --build

# 后续启动（无需重新构建）
docker compose up -d
```

等待约 30 秒，所有服务就绪后：

| 服务 | 地址 |
|------|------|
| 前端界面 | http://localhost:3000 |
| 后端 API | http://localhost:8000 |
| API 文档 | http://localhost:8000/docs |

### 停止服务

```bash
docker compose down        # 停止服务（保留数据）
docker compose down -v     # 停止服务并清除数据库（慎用）
```

### 查看日志

```bash
docker compose logs -f backend    # 后端日志
docker compose logs -f frontend   # 前端日志
docker compose logs -f            # 所有日志
```

## 2. 初始配置

### 2.1 首次访问

打开 http://localhost:3000，系统会自动进入 **设置向导**：

**第一步 — 选择 AI 供应商**

系统支持三个 AI 供应商，选择其中一个：

| 供应商 | 模型 | 适用场景 |
|--------|------|----------|
| OpenAI | GPT-4o | 英文为主的客户沟通 |
| Anthropic Claude | Claude 3.5 Sonnet | 多语言，理解力强 |
| Kimi (月之暗面) | Moonshot-v1-8k | 中文场景首选，价格实惠 |

**第二步 — 输入 API Key**

在对应供应商的官网获取 API Key：
- OpenAI: https://platform.openai.com/api-keys
- Anthropic: https://console.anthropic.com/settings/keys
- Kimi: https://platform.moonshot.cn/console/api-keys

填入 API Key，点击「下一步」。

**第三步 — 完成**

看到 "设置完成" 提示后，点击进入主界面。

### 2.2 导入 Facebook Cookies（必须）

> 这是最关键的步骤。系统需要你的 Facebook 登录状态才能搜索和发送消息。

1. 打开 Chrome 浏览器，登录你的 Facebook 账号
2. 安装 Chrome 扩展 **Cookie-Editor**（搜索 "Cookie-Editor chrome extension"）
3. 在 Facebook 页面点击 Cookie-Editor 图标
4. 点击「Export」→「Export as JSON」，复制全部内容
5. 进入 LeadFlow 的 **设置** 页面（侧边栏最后一项）
6. 在「Facebook 登录（Cookies）」区域，粘贴 cookies JSON
7. 点击「导入 Cookies」

看到 "已导入 XX 条 cookies" 提示即为成功。

## 3. 创建人设

> 人设定义了 AI 以什么身份、什么语气与潜在客户沟通。建议先创建人设，再创建获客任务。

### 3.1 进入人设管理

侧边栏点击「人设」→ 点击右上角「新建人设」。

### 3.2 填写人设信息

**人设名称**（必填）
- 示例：`专业商务顾问`、`友好销售代表`

**公司信息**
- 公司名称：如 `TechBridge`
- 公司描述：简述业务和优势
- 产品/服务：逐个输入并按回车添加，如 `AI 开发`、`自动化工具`

**销售代表信息**
- 姓名：AI 将以此身份自我介绍
- 职位：如 `商务经理`
- 性格特点：如 `热情、专业、善于倾听`

**对话风格**
- 语气风格：专业正式 / 专业友好 / 友好亲切 / 轻松随意
- 打招呼规则：定义首次消息的策略
  - 示例：`先赞美对方的职业成就，然后自然引出合作机会，不要直接推销`
- 对话规则：定义后续对话的策略
  - 示例：`注重建立关系，适时询问需求，遇到拒绝礼貌退出`

### 3.3 预览和保存

点击底部「系统提示词预览」查看 AI 将收到的完整指令，确认无误后点击「保存人设」。

### 3.4 管理人设

- **编辑**：在人设列表点击卡片进入编辑
- **设为默认**：编辑时勾选「设为默认人设」
- **删除**：编辑页右上角「删除」按钮，或列表卡片右上角垃圾桶图标

## 4. 创建并启动获客任务

### 4.1 创建任务

侧边栏点击「任务」→ 右上角「新建任务」。

**任务名称**（必填）
- 简明描述任务目的，如 `Facebook 外贸客户开发 - 东南亚`

**选择平台**
- 目前仅支持 Facebook（Twitter/Instagram 即将上线）

**搜索条件**
- 关键词（必填）：Facebook 搜索用的关键词
  - 示例：`外贸 B2B 供应商`、`跨境电商 深圳`、`SaaS founder`
- 地区：美国 / 英国 / 欧洲 / 东南亚 / 中东 / 全球
- 行业：如 `电子商务`、`新能源`、`SaaS`

**选择人设**
- 从下拉列表选择之前创建的人设
- 也可点击「+ 创建新人设」临时创建

**发送限额**
- 本次任务最多发送多少条消息（1-500）
- 建议首次使用设为 5-10 条，测试效果后再增加

点击「创建任务」。

### 4.2 启动任务

创建后任务状态为「草稿」。进入任务详情页，点击右上角「启动」按钮。

系统将自动执行以下流程：

```
搜索 Facebook 用户
    ↓
分析每个用户的个人资料
    ↓
AI 生成个性化首条消息
    ↓
模拟人类操作发送消息
    ↓
等待随机间隔（60-180秒）
    ↓
处理下一个用户
```

### 4.3 控制任务

| 按钮 | 作用 |
|------|------|
| 暂停 | 暂时停止发送，可随时继续 |
| 继续 | 恢复已暂停的任务 |
| 停止 | 终止任务，不可恢复 |

## 5. 监控任务进度

### 5.1 Dashboard

侧边栏点击「Dashboard」，查看全局概览：

- 总发送消息数
- 总线索数
- 活跃任务数
- 总任务数

### 5.2 任务详情

在任务列表点击任意任务名称进入详情页：

- **进度条**：已发送 / 限额
- **任务信息**：平台、关键词、地区、行业
- **线索列表**：本任务获取的所有线索及其状态

任务运行中时，页面每 5 秒自动刷新数据。

### 5.3 状态说明

**任务状态**

| 状态 | 含义 |
|------|------|
| 草稿 | 已创建未启动 |
| 运行中 | 正在自动执行 |
| 已暂停 | 手动暂停，可继续 |
| 已完成 | 达到发送限额 |
| 失败 | 手动停止或遇到错误 |

**线索状态**

| 状态 | 含义 |
|------|------|
| 已发现 | 搜索到但尚未分析 |
| 分析中 | AI 正在分析资料 |
| 已发送 | 消息已发送 |
| 已回复 | 对方已回复 |
| 已转化 | 标记为成功转化 |
| 失败 | 发送失败（主页私密等） |

## 7. 设置管理

侧边栏点击「设置」进入设置页面。

### 7.1 Facebook Cookies

- 导入/更新 Facebook 登录 cookies
- 查看当前 cookies 状态（已导入数量）
- 如果 cookies 过期（通常 1-2 周），需要重新导入

### 7.2 AI 供应商

- 切换 AI 供应商（OpenAI / Anthropic / Kimi）
- 更新 API Key
- 设置 API Base URL（用于 OpenAI 兼容的第三方 API）
- 设置代理地址（可选，用于网络受限环境）

### 7.3 发送参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 发送间隔（最小） | 60 秒 | 两条消息之间的最短等待时间 |
| 发送间隔（最大） | 180 秒 | 两条消息之间的最长等待时间 |
| 每日最大发送量 | 50 条 | 单日发送上限，防止账号被封 |

**安全建议**：
- 发送间隔不要低于 60 秒
- 每日发送量不要超过 50 条
- 新账号建议从 10-20 条/天开始

### 7.4 退出登录

点击侧边栏底部「退出登录」按钮。

## 8. MCP 自然语言控制

LeadFlow 集成了 MCP (Model Context Protocol) 服务，可通过 AI 助手（如 Claude Code / OpenClaw）用自然语言控制系统。

### 使用示例

```
"搜索 Facebook 上深圳的跨境电商卖家，发送 10 条问候消息"
"暂停正在运行的任务"
"查看当前任务状态"
"列出所有线索"
```

### 启动 MCP 服务

```bash
docker compose up -d mcp-server  # 如果 docker-compose 中有配置
# 或
cd mcp-server && python server.py
```

## 9. 常见问题

### Q: 打开页面显示空白或加载中

**A**: 检查后端是否正常运行：
```bash
curl http://localhost:8000/health
```
如果不通，重启服务：
```bash
docker compose restart backend
```

### Q: 任务启动后没有反应

**A**: 可能的原因：
1. Facebook Cookies 过期 → 重新导入 cookies
2. AI API Key 无效 → 在设置页面检查并更新
3. 网络问题 → 检查是否需要配置代理

查看后端日志定位问题：
```bash
docker compose logs -f backend
```

### Q: 消息发送频率太快/太慢

**A**: 在设置页面调整「发送间隔」参数。建议保持至少 60 秒的最小间隔。

### Q: Facebook 账号被限制

**A**:
1. 立即停止所有任务
2. 增大发送间隔（如最小 120 秒，最大 300 秒）
3. 降低每日发送量（如 10-20 条）
4. 等待 24-48 小时后再恢复使用
5. 考虑使用代理 IP

### Q: Cookies 多久需要更新一次

**A**: Facebook Cookies 通常 1-2 周过期。如果任务中出现登录相关错误，说明需要重新导入。

### Q: 如何备份数据

**A**: 数据存储在 Docker 卷中，可通过以下方式备份：
```bash
# 备份数据库
docker compose exec postgres pg_dump -U leadflow leadflow > backup.sql

# 恢复数据库
cat backup.sql | docker compose exec -T postgres psql -U leadflow leadflow
```

### Q: 如何完全重置系统

**A**:
```bash
docker compose down -v    # 删除所有容器和数据卷
docker compose up -d --build  # 重新构建并启动
```
然后重新走初始配置流程。
